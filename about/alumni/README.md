# 🎓 Alumni Directory (`/about/alumni`)

Japan Snowflake User Group (SnowVillage) の土台を築き、過去に多大な貢献をしていただいた元運営メンバー（卒業生）の殿堂入り（アーカイブ）ページです。

## 📄 ファイルと構造の解説

### `index.html`
このファイルは、ルートディレクトリから見て2階層下（`ルート > about > alumni`）に位置しているため、パスの指定方法が他のページと異なります。

#### 1. 階層を意識したパス指定（重要）
共通のスタイルシートや画像、JavaScriptを読み込むため、すべての相対パスに `../../`（2つ上の階層へ戻る）が使用されています。
- CSS: `href="../../style.css"`
- 画像: `src="../../images/alumni/ファイル名.jpg"`
- JS: `src="../../script.js"`

#### 2. Component Injection (動的生成)
ページ末尾のスクリプトにおいて、`renderHeader('../..', 'about')` と指定しています。
- 第1引数 `../..`: 共通ヘッダー内のロゴやリンクが、2階層下からでも正しくルート階層を向くように補正します。
- 第2引数 `'about'`: グローバルナビゲーションの「運営について」をアクティブ（青文字）に維持します。

#### 3. 卒業生（Alumni）専用のUIデザイン
現役の運営メンバー（Mayors）のカードと比較して、視覚的に「殿堂入り」や「過去の功労者」であることを表現するための微調整がインラインスタイルで行われています。
- **`.card.member-card` のインラインスタイル**: 
  `background-color: #f8fafc; border: none;` を指定することで、通常時のパキッとした白背景＋枠線ではなく、少し背景に溶け込むような柔らかいグレー背景のカードにしています。
- **在籍期間の明記**: 
  現役メンバーにはない「📅 在籍期間（Tenure）」の項目が追加されています。

## 💡 運用・更新のポイント

### 卒業生の追加方法
運営メンバーが卒業（Alumniへ移行）する際は、以下の手順で追加してください。

1. **画像の移動**: そのメンバーの画像を `/images/organizers/` から `/images/alumni/` へ移動させます。
2. **HTMLのコピー**: 既存の `.card.member-card` ブロックをコピーし、グリッド内に追加します。
3. **情報の書き換え**: 
   - 画像パス: `../../images/alumni/新しい画像.jpg`
   - 名前、在籍期間、当時の役割（Former Tech Lead等）、および感謝を込めた貢献内容のコメントを記載してください。
