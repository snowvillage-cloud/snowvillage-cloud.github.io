# 👥 About Organizers (`/about`)

SnowVillageを支える現役運営メンバー（Mayors）、協賛者（Sponsors）、および卒業生（Alumni）を紹介するディレクトリです。

## 🏗 新しい動的レンダリング・アーキテクチャ
HTMLの肥大化を防ぎ、運用（メンバーの追加・卒業・情報の更新）を安全かつ簡単に行うため、**「データと表示（ビュー）の分離」**を行なっています。

### 📄 ファイル構成と役割

- **`aboutData.js`**: 
  **【重要】** 運営メンバーとスポンサーの全データを管理するファイルです。名前、肩書き、画像パス、SNSリンクのURLなどをJSON形式の配列（`organizersData`, `sponsorsData`）として保持しています。
- **`index.html`**: 
  UIの枠組み（コンテナ）のみを提供するファイルです。メンバーを表示する場所には空の `<div id="organizers-grid"></div>` などを配置し、ページ末尾の `<script>` でデータをHTMLに変換して流し込みます。
- **`/alumni/`**:
  卒業生を紹介する下層ディレクトリです。（※こちらは独自の `index.html` を持っています）

### ⚙️ 技術的な工夫点（動的生成のメリット）
1. **インラインSVGの隠蔽**: 
   X（旧Twitter）やLinkedInの複雑なSVGアイコンコードを、JSのテンプレートリテラル内に閉じ込めることで、HTML全体を数百行単位で軽量化しています。
2. **アニメーションの自動計算**: 
   カードが順番にフワッと表示されるエフェクト（`.reveal`）のタイミングを、配列のインデックスを用いて `${index * 0.1}s` と自動計算させています。何人増えても手動でディレイ（遅延）を設定する必要がありません。

## 💡 運用・更新のポイント

### メンバー・スポンサーの追加と編集
HTMLファイル（`index.html`）を編集する必要はありません。**`aboutData.js` を開いてデータを編集するだけ**で、自動的にサイトに反映されます。

**▼ 追加例（`aboutData.js`内）**
```javascript
{
  name: "新しいメンバーの名前",
  role: "役割・役職",
  desc: "自己紹介文や所属",
  photo: "../images/organizers/新規画像ファイル.jpg",
  xUrl: "[https://x.com/](https://x.com/)...", // 不要な場合は空文字("")にすればアイコンは出ません
  linkedInUrl: "[https://www.linkedin.com/](https://www.linkedin.com/)..." 
}
```

- ※SNSのリンクURLが存在する場合（空でない場合）のみ、自動的にアイコン付きのリンクボタンが生成される安全なロジックになっています。
