# 📖 Community Guide (`/guide`)

SnowVillageを120%楽しむための「歩き方」ガイドや、Slackの利用ルール、Snowflakeオフィスを利用したイベント開催のノウハウなどをまとめたディレクトリです。
新規参加者がコミュニティに溶け込むための重要なオンボーディング・コンテンツとなります。

## 📄 ファイルと構造の解説

### `index.html`
長文のガイドラインをユーザーが飽きずに読めるよう、多彩なUIコンポーネントを組み合わせて構造化しています。

#### 主要なUIコンポーネントとCSSクラス
- **`.guide-section`**: 
  大きなトピック（Mayorsについて、Slackの歩き方、オフィス利用ルール等）ごとに区切るセクションコンポーネントです。
- **`.guide-block`**: 
  セクション内の具体的な説明文を囲むカード型のブロックです。適度な余白と背景色により、文章の塊を視覚的に整理しています。
- **`.callout` (Info / Warning / Tip)**: 
  特に注目してほしい補足情報や注意点をハイライトするためのコールアウト・ボックスです。
  - `info`: 豆知識や案内（青系）
  - `warning`: 重要な注意書きやルール（黄色/赤系）
  - `tip`: 便利な使い方やノウハウ（緑系）
- **テキスト装飾タグ**: 
  - `.channel-tag`: `#general` のように、Slackのチャンネル名を目立たせるタグ風の装飾。
  - `.mention-tag`: `@mayors` のように、メンションを視覚的に分かりやすくする装飾。

#### Component Injection (動的生成)
- ページ末尾の `<script>` 内で `renderHeader("..", "guide")` を実行することで、ナビゲーションメニューの「Guide」がアクティブ（青文字）になり、相対パスが正しく解決されたヘッダーが読み込まれます。

## 💡 運用・更新のポイント
- **ルールの追加・変更**: 
  新しいSlackの運用ルールや、オフィスの入館ルールに変更があった場合は、該当する `.guide-block` 内のテキストを直接編集してください。
- **可読性の維持**: 
  文章が長くなりすぎる場合は、適宜 `<h3>` や `<h4>` で見出しを作り、`.callout` を使って要点を分離することで、スマートフォンの画面でも読みやすいレイアウトを維持できます。
