# 🤝 Slack Join & Code of Conduct (`/join`)

SnowVillageの公式Slackコミュニティへの参加導線となるディレクトリです。
コミュニティの心理的安全性と質を担保するため、参加前に必ず「行動規範（Code of Conduct）」を読み、同意していただくためのゲートウェイ（関所）として機能します。

## 📄 ファイルと実装ロジックの解説

### `index.html`
このページは、利用規約の提示と同意の取得を行うための特別なUI構造を持っています。

#### 1. ヘッダー・フッターの独立設計（他ページとの違い）
他のページ（about, news等）では `script.js` の `renderHeader()` を用いて動的にヘッダーを生成していますが、このページは `<header>` および `<footer>` を**HTML内に直接ハードコード（直書き）**しています。
- **意図**: 参加前のユーザーに対するランディングページとしての独立性を高める、あるいは同意画面という特殊なページ性質上、共通スクリプトの実行依存を減らすためのフェイルセーフ設計です。

#### 2. 行動規範のUIコンポーネント
- **`.rules-box`**: 
  行動規範全体を囲むボックスです。長文でも圧迫感を与えないようデザインされています。
- **`.rule-item`**: 
  「オープンなコミュニケーション」「データを大切に」など、各ルールを項目ごとに区切るクラスです。

#### 3. 同意ゲーティング・システム（JavaScript連携用）
ユーザーが規約を読まずにSlackに参加してしまうのを防ぐための仕組みが実装されています。
- **`.agreement-area`**: 同意チェックボックスと参加ボタンを配置するエリア。
- **カスタムチェックボックス**: 
  `<label class="checkbox-container">` と `<span class="checkmark"></span>` を用いて、ブラウザ標準のチェックボックスではなく、サイトのデザインに馴染むオリジナルのチェックボックスUIを構築しています。
- **ボタンの制御 (`#slack-join-btn` / `.disabled`)**: 
  Slackへの参加ボタンは、初期状態で `.disabled` クラスが付与され、クリックできない（または無効化された見た目）になっています。
  （※通常、`script.js` 側で `#agree-checkbox` の状態を監視し、チェックが入った時のみ `.disabled` クラスを外してリンクを有効化するJavaScriptロジックと連動して機能します。）

## 💡 運用・メンテナンスの重要ポイント

### 1. Slack招待リンクの更新
Slackの招待リンク（Shared Invite Link）は、設定によって**有効期限が切れる**ことがあります。
リンクを更新する際は、`index.html` 内の以下の箇所の `href` 属性を新しいものに差し替えてください。

<a href="[https://join.slack.com/t/snowvillage/shared_invite/](https://join.slack.com/t/snowvillage/shared_invite/)..." id="slack-join-btn">
